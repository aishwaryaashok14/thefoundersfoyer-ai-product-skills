# Behavioral Product Design

A Claude Code skill that applies behavioral science to your product decisions — using Kristen Berman's 3 Bs framework from Irrational Labs (co-founded with Dan Ariely).

## When to use this skill

Use this when:
- **Users aren't doing what you designed for** — they sign up but don't activate, browse but don't buy, start but don't finish
- **You're about to build a highly-requested feature** — the #1 feature request might destroy your product (it happened to Gas App)
- **You're designing a signup flow, onboarding, or conversion funnel** — even removing one text field can lift conversion 40%
- **User interviews are contradicting user behavior** — people say one thing and do another
- **You're optimizing for "engagement" or "retention"** — those aren't behaviors, and this skill will make you define what is
- **You're in a "hot state" domain** (social, impulse purchases, emotional decisions) where user self-reports are structurally unreliable

Don't use this when:
- You don't have users yet — you need to ship first (use Vibe-to-Ship Coach)
- The problem is that users can't understand what you do — that's positioning, not behavior

## What it does

Walks you through the 3 Bs framework:

1. **B1 — Behavior** — Forces you to define the uncomfortably specific action you want to change (not an outcome like "retention" — a literal action a camera could record)
2. **B2 — Barriers** — Maps every step between the user and the behavior, separating logistical barriers (steps, time, complexity) from psychological barriers (uncertainty, fear, cognitive load)
3. **B3 — Benefits** — Redesigns motivation around immediate rewards, not future payoffs

Also includes:
- **Hot/Cold State Diagnostic** — Calibrates how much to trust user feedback based on the decision domain
- **Step-Mapping Exercise** — Maps real-world steps including those outside your product
- **Pre-Testing Methodology** — 4 conditions, ~1,000 users, relative comparison
- **7 cautionary case studies** — Netflix, ChatGPT, Gas App DMs, TikTok misinformation, FinTech budgeting, SMB incentives, signup field removal
