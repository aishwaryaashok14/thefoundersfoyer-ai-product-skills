# Vibe-to-Ship Coach

A Claude Code skill that takes you from messy idea to working prototype in the fastest path possible — using Colin Matthews' step-by-step AI prototyping process, Eric Simons' speed thesis from Bolt, and Dan Shipper's creative momentum techniques.

## When to use this skill

Use this when:
- **You haven't shipped yet** and you've been "planning" for more than a week
- **You have a messy idea** and don't know where to start building
- **You want to test an idea with a real prototype** instead of a slide deck or mockup
- **You're choosing between AI coding tools** (Bolt vs. Lovable vs. Replit vs. Cursor vs. V0)
- **You're stuck in a debugging loop** with an AI coding tool and nothing is working
- **You're non-technical** but want to build something real
- **You're over-planning** — writing PRDs, drawing wireframes, debating architecture — when you should be building

Don't use this when:
- You're building production infrastructure — this is for prototypes and MVPs
- You need to maintain an existing large codebase — use Cursor directly
- You're not sure what to build — use Rigorous Thinking Coach to clarify first

## What it does

Walks you through 8 steps from idea to shipped prototype:

1. **Lower the Stakes** (Dan Shipper) — Dissolves perfectionism and overthinking
2. **Extract the Idea** — Gets your concept down to one sentence in 60 seconds
3. **Pick the Right Tool** — Colin's tool selection matrix based on your situation
4. **Generate a Lightweight PRD** — One page, not twenty
5. **Plan Before Code** — "Make a plan. Start with the data model. Don't write any code."
6. **Build Phase by Phase** — Frontend-first with mock data, one phase at a time
7. **Debug Smart** — The "don't write any code" method: explain first, fix second, never click auto-fix
8. **Ship Immediately** — Put the real app in front of people and get feedback

Includes the tool selection matrix:

| Situation | Tool |
|---|---|
| Quick prototype, general purpose | Bolt |
| Non-technical, real project | Lovable |
| Technical, wants control | Replit |
| Beautiful UI fast | V0 |
| Refining existing code | Cursor |
